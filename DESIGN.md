# Gesso Grammar

This is the **grammar layer**: brand-neutral rules for building UI that reads as
intentional and coherent, whatever the project. It is the versatile base — it
must work equally well for a marketing page, a dense dashboard, and a consumer
app. It carries no palette personality, no imagery, no voice. Those live in a
**skin** (see `skins/`), which overrides these tokens with project-specific
values and sits on top of this grammar.

Two layers, never fused:

- **Grammar** (this file) — *how* good design behaves. Scale, hierarchy,
  discipline, motion. Stable across every project.
- **Skin** — *who* the project is. Palette, accent, typeface, imagery, tone.
  Different every project. One ships as reference: `skins/default`.

> Shipping a single locked look makes every app look the same (the shadcn fate).
> Gesso ships the grammar plus one reference skin, and documents how to write
> your own. The brief→skin engine and the visual self-verify loop are v1 — see
> the repo README roadmap. Not in this foundation.

## Overview

Agents converge on the most statistically conventional UI by default, so output
looks generic and interchangeable. Gesso counters that by giving the agent an
explicit, opinionated grammar to consult on every interaction — the same role
AGENTS.md plays for code conventions, DESIGN.md plays for visual ones.

Core principles, in priority order:

1. **Hierarchy comes from size, weight, and tone — not color.** Most text is one
   of three neutral tones (`fg`, `fgMuted`, `fgSubtle`). Reach for a bigger size
   or heavier weight before you reach for a color.
2. **Borders carry structure; shadows are atmosphere.** A 1px hairline defines
   almost every inline surface. Shadows are reserved for things that genuinely
   float.
3. **One accent, used sparingly.** The grammar permits exactly one hero color
   (the accent slot). It marks the single most important action or datum on a
   view — it is not a decoration sprayed across surfaces.
4. **A closed scale beats a free hand.** Pick spacing, radius, and type from the
   defined steps. In-between values are the texture of generic UI.
5. **Restraint in motion.** Short, eased, purposeful. Never bouncy, never
   attention-seeking.

## Colors

The foundation is a **neutral gray ramp** plus **near-black ink** on white. The
ramp is intentionally temperature-free; warmth or coolness is something a skin
adds, not the grammar.

- **Backgrounds** — `bg` (page), `bgElev` (raised card/popover), `bgSunken`
  (inset/sidebar), `bgTint` (neutral hover). Keep the steps subtle; the
  difference between surfaces should be felt more than seen.
- **Text tones** — `fg` / `fgMuted` / `fgSubtle`. Three tones is enough for full
  hierarchy. Avoid alpha-on-text for hierarchy; use the ramp.
- **Brand ink** — `brand` is the silhouette color: primary text and the primary
  CTA fill. Near-black by default; a skin may deepen or shift it.
- **Accent** — `accent` is the one hero color (see Do's and Don'ts). `accentBg`
  is its soft tinted surface; `accentFg` is text on a filled accent.
- **Semantic** — `success` / `warning` / `danger` / `info` are status, not
  brand. Keep them muted; an error is information, not an alarm.
- **Data-viz** — `viz1…viz6` are a categorical series palette so charts read as
  one family. `viz1` tracks the accent by convention. (Sequential and diverging
  ramps and color-blind safety are an open design task — see docs/gaps.md.)

All color/text component pairs target **WCAG AA (4.5:1)** minimum. The DESIGN.md
linter enforces this — see the repo README for wiring.

## Typography

One sans family does almost all the work. A skin may add a serif for editorial
moments, but the product UI is sans.

- **Scale** — `display`, `h1`–`h4`, `bodyLarge`, `body`, `bodySmall`, `caption`,
  `mono`. That is the whole set; don't introduce sizes between steps.
- **Weight** — Regular 400, Medium 500, Semibold 600. Bold 700 is rare and
  deliberate. Build emphasis from weight before color.
- **Tracking tightens as size grows** — display and large headings use
  `-0.02em`; body sits at 0; small overlines open up slightly.
- **Hierarchy = size + weight + tone.** A muted 13px caption and a 600-weight
  20px h4 carry a whole interface without a single accent color.

## Layout

- **Closed spacing scale**, 4px base. Compose rhythm from `--sp-*` steps only.
- **Nesting radius** — each layer is ~one radius step tighter than its parent: a
  pill button inside an `r-lg` card inside the page. Don't mix unrelated radii
  on one surface.
- **Density is deliberate.** Marketing surfaces breathe (generous vertical
  rhythm, `--sp-16`+ between sections). Dashboards tighten. The dense-table mode
  exists for data-heavy views — use it instead of cramming the default table.
- **Generous side padding** on primary content panes at desktop (≥`--sp-12`).

## Elevation & Depth

Two coexisting systems, and the rule for choosing:

- **Hairline rings** for inline surfaces (cards on the canvas, list rows): a 1px
  `border`, **no shadow**.
- **Soft elevation** for floating chrome only (popover, menu, sheet, sticky
  composer): a low blurred shadow plus a 1px ring (`--shadow-md` / `--shadow-lg`
  / `--shadow-pop`). The shadow should read as atmospheric haze, not a hard drop.

Never use bright or colored shadows. Never use a shadow to fake importance on an
inline card — raise it with hierarchy, not elevation.

## Shapes

- **Radius** — `sm` chips, `md` inputs/buttons, `lg` cards, `xl` large shells,
  `pill` for pills/avatars.
- **Borders** — 1px hairline in `border`; `borderStrong` for an emphasized edge;
  `divider` for full-bleed section rules. The border, not a fill, is the default
  way to separate things.
- **Anti-pattern: never use a left-border accent stripe** to mark active/selected
  state. Use a tint (`bgTint`) instead.

## Components

These are grammar-level defaults — structure and discipline, not final styling.
A skin restyles them via the token values; their *shape* is grammar.

- **Button (primary)** — solid `brand`, `brandFg` text, `r-md` (or `pill` for
  marketing). One primary action per view.
- **Button (accent)** — solid `accent`. Use only when the primary action is also
  the hero action; don't stack accent and brand fills next to each other.
- **Button (secondary)** — `bgElev` fill, hairline `border`, same radius.
- **Card** — `bgElev`, hairline `border`, `r-lg`, `--sp-5` padding, no shadow.
- **Input** — `bgElev`, hairline `border`, `r-md`; focus shows the `ring`, not a
  heavy glow.
- **Table** — comfortable by default; switch to dense mode (`--table-row-h-dense`,
  tighter cell padding) for data surfaces. Header is `bgSunken` / `fgMuted`,
  uppercase optional. Zebra striping (`--table-stripe`) is nearly invisible by
  design; prefer row borders.
- **Chart** — categorical series draw from `viz1…viz6` in order; gridlines stay
  quieter than data (`--viz-grid`); one series may take the accent to lead.

## Do's and Don'ts

The grammar's soul lives here — the spec has no fields for motion, anti-patterns,
or accent discipline, so they're stated as rules.

**Do**
- Build hierarchy with size, weight, and the three text tones first.
- Let the 1px border carry structure; add a shadow only when something floats.
- Spend the accent on one thing per view: the key action, focus, or lead datum.
- Step radius down one notch per nesting layer.
- Keep motion short (`--dur-fast`/`--dur-med`), eased (`--ease-out`), purposeful.
- Pick every size, space, and radius from the scale.

**Don't**
- Don't carry hierarchy with color, or spray the accent across surfaces.
- Don't add shadows to inline cards, or use colored/harsh shadows.
- Don't use a left-border accent stripe for selected state — tint the row.
- Don't invent in-between spacing, radius, or type sizes.
- Don't use bouncy springs, parallax, scroll-jacking, or decorative motion.
- Don't reach for emoji as iconography.
- Don't bolt a bluish-purple "AI gradient" onto everything — depth comes from
  the neutral ramp and hairlines, not gradients.

# Studio Skin

The warm, photographic counterpart to the default skin. Same locked grammar —
spacing, radius, hierarchy, motion — but it claims two identity slots the
default declines: a **hero accent** and a **photographic backdrop**.

## Overview

Golden-hour light, off-white paper, an editorial serif, and one warm terracotta.
Built for premium marketing, sign-in, and hero surfaces where atmosphere matters.

## Colors

- **Warm paper** surfaces (`bg` `#F7F1E8`), deep warm ink (`fg` `#221A12`).
- **Claimed accent** — terracotta `#C5613B`, spent on the one primary action,
  focus ring, and a lead datum. (The default skin opts out; Studio opts in — both
  are valid uses of the same grammar slot.)
- **Earthen data-viz** family so charts feel native to the palette.

## Typography

- **Instrument Serif** at display scale — editorial, warm. **Inter** for all UI.
- Hierarchy is still size + weight + tone; the serif is identity, not structure.

## Layout & Depth (backdrop + glass)

Studio uses the grammar's **backdrop slot** (`--backdrop`) for a golden-hour wash
and the **glass pattern** (`.glass`, `--glass-*`) for frosted cards that float on
it. Default backdrop is procedural (portable). For a photographic backdrop, point
the slot at the asset library:

```css
--backdrop: url('assets/backgrounds/warm-goldenhour-01.jpg') center/cover;
```

Glass is for surfaces **over a backdrop only**. On flat surfaces keep the hairline
card — the grammar's elevation rule still holds.

## Do's and Don'ts

**Do** spend the terracotta on one thing per view; use the serif only at display
scale; reserve glass for backdrop surfaces.
**Don't** put glass on flat white panels; don't let the accent spread; don't set
body text in the serif.
